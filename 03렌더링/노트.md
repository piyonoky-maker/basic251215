## 리액트와 리렌더링

: 리액트의 장점이 리렌더링에 있다

### 실습1 - render02.html

바닐라JS -> 변경으로 인해 Element를 다시 그림
비교 대상 - render03.html<br>
리액트 -> 변경된 부분만 다시 그림 - 효과적임<br>

이런게 가능한 이유?<br>
바뀐 걸 알려면 계속 비교를 해야 한다. -> trade off ( 좋은게 있으면 나쁜것도 있다 )
<br>

React앨리먼트는 불변객체(immutable) 이다
불변객체: 변하지 않는 객체임

ReactDOM.render(element, root) 전달할 뿐
변경 판단 및 바영은 리액트가 알아서 한다

변경사항에 대한 반영은 리액트에게 일임함

엘리먼트를 생성한 이후에는 해당 엘리먼트의 자식이나
속성을 변경할 수 없습니다.

리액트는 비교알고리즘을 가지고 있다
엘리먼트 타입이 바뀌면?
이전 엘리먼트는 버리고 새로 그린다.
만일 반대로 타입이 같으면..

key를 번저 비교하고
props를 비교해서 변경사항을 반영한다

## 재조정

DOM이 계속 들어오면 이전 버츄얼 DOM과 한번 더 비교 하고 그것을 반영한다
같은 타입의 두 React DOM 엘리먼트를 비교할 떄,
React는 두 엘리먼트의 속성을 화인하여,
동일한 내역은 유지하고 변경된 속성(:props를 말함)들만 갱신한다
